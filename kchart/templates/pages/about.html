kchart.io is a Korean music service chart aggregator. Currently, the realtime top 100 charts for the four largest Korean digital music services (Melon, Genie, Bugs, Mnet) are indexed.

FAQ

How does the aggregated chart scoring work?

kchart.io's overall chart scoring is similar to instiz. Each service's charts are weighted based on their relative userbase sizes (i.e. being rated #1 on Melon is worth more that being rated #1 on Genie)

Is historical chart data available?

Yes, except for Melon. All the chart services except melon provide historical data, and eventually all of the publically available historical data for Genie, Bugs and Mnet will be made available on kchart.io, but it will take some time for the backlog scripts to complete.

Why are <particular service's> charts unavailable for certain timestamps?

Charts are indexed via web scraping. Missing chart data most likely means the kchart.io scraper was unable to connect to the chart service's website for some reason, but missing data will eventually be filled in by the backlog scripts.

How is kchart.io any different than instiz?

It's not, and intiz's data is almost certainly more reliable. However, instiz's interface for accessing historical chart data is slow and painful to use. kchart.io's intended purpose is to make it easy to see a particular song's chart history (i.e. where and when the song peaked on each chart, along with some other historical trend data).

Is there an API available?

Yes, although it is still in development and should not be considered stable. Currently the available API endpoints are:
  • /api/v1/charts/realtime/
  • /api/v1/charts/realtime/melon|genie|bugs|mnet/
  • /api/v1/songs/song_id/
Timestamps in the form of YYYYmmddHH can be appended to the chart url endpoints to retrieve a specific historical chart. The extra field in the songs api can be used to specify which chart data to include in the song details.
